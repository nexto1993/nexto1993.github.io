üîë Key Concepts Explained

1. What are Indexers?

Definition: Indexers allow objects to be indexed like arrays using square bracket notation.

  • Enable object[index] syntax for custom classes
  • Provide controlled access to internal collections or data
  • Similar to properties but use square brackets instead of dot notation
  • Can have both getter and setter accessors
2. Indexer Syntax
Basic Indexer Declaration:
public ReturnType this[IndexType index] { get; set; }
  • this keyword indicates it's an indexer
  • Square brackets contain the index parameter(s)
  • Can have multiple parameters for multi-dimensional access
  • Index parameters can be of any type (int, string, etc.)
3. Getter and Setter

Definition: Indexers can provide both read and write access to data.

  • Getter: Returns value when accessing object[index]
  • Setter: Assigns value when using object[index] = value
  • Can be read-only (getter only) or write-only (setter only)
  • Allows validation and custom logic during access
4. IP Address Example Benefits

Real-world Application: IP class encapsulates an IP address with convenient access.

  • Encapsulation: Internal array is private, access is controlled
  • Intuitive API: ip[0] is more readable than ip.GetSegment(0)
  • Validation potential: Can add bounds checking in getter/setter
  • Formatted output: Address property provides dot-separated string

// 2D indexer example public int this[int row, int col] { get { return matrix[row, col]; } set { matrix[row, col] = value; } } // String indexer example public object this[string key] { get { return dictionary[key]; } set { dictionary[key] = value; } }

üí° Key Learning Points

Important Indexer Concepts:

  • Array-like syntax: Makes objects behave like collections
  • Encapsulation: Provides controlled access to internal data
  • Type flexibility: Index parameters can be any type
  • Multiple parameters: Support multi-dimensional access patterns
  • Validation opportunity: Add bounds checking and error handling
  • IntelliSense support: IDE provides autocompletion and type checking

üîß Potential Improvements

The current IP class could be enhanced by:

  • Bounds checking: Validate index is between 0-3
  • Value validation: Ensure segments are between 0-255
  • Exception handling: Throw meaningful exceptions for invalid access
  • Read-only indexer: Consider making it read-only if modification isn't needed
  • Fix typo: "segemnts" should be "segments"
  • ToString() override: Override ToString() to return formatted address

üîß Improved Implementation Example

public class ImprovedIP { private int[] segments = new int[4]; public int this[int index] { get { if (index < 0 || index > 3) throw new ArgumentOutOfRangeException("Index must be 0-3"); return segments[index]; } set { if (index < 0 || index > 3) throw new ArgumentOutOfRangeException("Index must be 0-3"); if (value < 0 || value > 255) throw new ArgumentOutOfRangeException("Value must be 0-255"); segments[index] = value; } } }

üöÄ Practical Applications

Indexers are commonly used in:

  • Collections: Custom list or dictionary implementations
  • Matrix/Grid classes: 2D or multi-dimensional data access
  • Configuration objects: Key-value pair access
  • Database rows: Column access by index or name
  • File systems: Directory/file access by path
  • Game development: Board positions, inventory slots
