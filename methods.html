üîë Key Concepts Explained

1. Void Methods

Definition: Methods that perform actions but don't return a value.

public void print()
  • Use void keyword as return type
  • No return statement needed (optional for early exit)
  • Perfect for actions like printing, logging, or modifying object state
2. Methods with Return Values

Definition: Methods that calculate and return a value to the caller.

public int MethodReturn()
  • Must specify return type (int, string, bool, etc.)
  • Must include return statement
  • Returned value can be assigned to variables or used in expressions
3. Method Parameters

Definition: Values passed to methods to customize their behavior.

public void MethodWithParams(int x)
  • Value parameters: Copy of the value is passed
  • Changes to parameters don't affect original variables
  • Can have multiple parameters separated by commas
4. Ref Parameters

Definition: Parameters that allow methods to modify the original variable.

public void ChangeValue(ref int x)
  • Variable must be initialized before passing
  • Changes inside method affect the original variable
  • Must use ref keyword in both method signature and call
  • Passes reference to memory location, not a copy
5. Out Parameters

Definition: Parameters used to return multiple values from a method.

public void OutKeyword(out int age)
  • Variable doesn't need to be initialized before passing
  • Method must assign a value to out parameter
  • Must use out keyword in both method signature and call
  • Useful for methods that need to return multiple values
6. Method Overloading

Definition: Multiple methods with the same name but different parameter lists.

public void doMath(int x, int y)
public void doMath(double x, int y)
  • Same method name, different parameters (type, number, or order)
  • Compiler determines which method to call based on arguments
  • Return type alone cannot differentiate overloaded methods
  • Provides flexibility and intuitive API design
7. Expression-Bodied Methods

Definition: Concise syntax for simple methods using lambda expressions.

public int returnValue(int x) => x + 1;
  • Introduced in C# 6.0
  • Perfect for simple, one-line methods
  • More readable than traditional method syntax
  • Automatically returns the expression result

üí° Key Learning Points

Important Method Concepts:

  • Method signature: Combination of method name and parameter list
  • Return type: Use void for no return, specific type for return value
  • Parameter passing: Value (copy), ref (reference), out (output)
  • Method overloading: Same name, different parameters for flexibility
  • Expression syntax: Use => for simple, one-line methods

üöÄ Practical Applications

Method concepts demonstrated here are essential for:

  • API Design: Creating intuitive method signatures
  • Data Processing: Transform input data and return results
  • Object Manipulation: Modify object state through methods
  • Mathematical Operations: Overloaded methods for different numeric types
  • Validation Logic: Methods that check and return validation results
  • Factory Methods: Methods that create and return new objects
