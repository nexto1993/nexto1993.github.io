Field & Constant in C#

Understanding fields, constants, and static members in C# classes

Code Analysis

Employee Class (Employee.cs)

namespace Field_Constant { public class Employee { // Constants - cannot be changed after compilation public const double TAX = 0.05; // Static field - shared among all instances, can be modified public static double Changeable_TAX = 0.05; // Instance fields - unique to each object instance public string FirstName; public string LastName; public double Salary; public double LoggedHours; } }

Program Class (Program.cs)

namespace Field_Constant { internal class Program { static void Main(string[] args) { // Create an instance of Employee Employee employee = new Employee(); // Set instance field values employee.FirstName = "Adel"; employee.LastName = "Alosaimi"; employee.Salary = 500; employee.LoggedHours = 40; // Modify static field (affects all instances) Employee.Changeable_TAX = 100; // Calculate net salary using constant var netSalary = employee.Salary * 1000 - Employee.TAX; Console.WriteLine("Hello, World!"); } } }

üîë Key Concepts Explained

1. Constants (const)

Definition: Constants are compile-time values that cannot be changed once defined.

  • Declared with the const keyword
  • Must be initialized at declaration
  • Value is embedded in the compiled code
  • Accessed through the class name (e.g., Employee.TAX)
  • Implicitly static
2. Static Fields

Definition: Static fields belong to the class itself rather than any specific instance.

  • Declared with the static keyword
  • Shared among all instances of the class
  • Can be modified at runtime
  • Accessed through the class name (e.g., Employee.Changeable_TAX)
  • Initialized when the class is first loaded
3. Instance Fields

Definition: Instance fields are unique to each object instance.

  • Each object has its own copy
  • Can be modified through the object instance
  • Accessed through object references (e.g., employee.FirstName)
  • Initialized when the object is created

üí° Key Learning Points

Important Notes:

  • Constants are compile-time: The value 0.05 for TAX is burned into the compiled code
  • Static fields are runtime: Changeable_TAX can be modified during program execution
  • Memory efficiency: Static members are shared, saving memory when you have many instances
  • Access patterns: Use class name for static/const, use instance for regular fields

üöÄ Practical Usage

In the example code:

  • TAX is a constant representing a fixed tax rate that never changes
  • Changeable_TAX is a static field that could represent a configurable tax rate
  • Employee details like FirstName, LastName, etc., are instance fields unique to each employee
